Audrey Link Relations

"rename" rel

The "rename" relation is used to indicate a link from a collection to an endpoint for POSTing a request to rename a child resource. Such a link will only exist for collections that allow explicit control over child naming.

Requests to a collection's "rename" endpoint should use the POST method and the application/json content type. The body of the request should be a JSON object with the keys: "from_name" and "to_name" whose values are strings indicating the old and new names. For example: 

{
    "from_name": "foo",
    "to_name": "bar"
}

Responses described below...

Status Notes
204 No Content The rename was successful. The response includes Location and Content-Location headers with the url of the renamed child resource.
400 Bad Request The rename was not allowed (non-unique or invalid new name). The body of the response is a JSON object with an "error" property explaining the reason.
404 Not Found No child object exists with the value of "from_name".
