Audrey Link Relations

"upload" rel

The "upload" relation is used to indicate a link from the root resource to an endpoint for POSTing a request to upload one or more files.

Requests to the "upload" endpoint should use the POST method and the multipart/form-data content type. The request should contain any number of file uploads using the parameter names of your choice. The server will create a new file in the GridFS for each uploaded file.

The server will return a JSON document with the parameter names from the request as the keys. Each value will be a JSON object with the parameter "FileId" whose value is a string representation of the file's ObjectId.

Example: You POST a request with two files identified by the parameters "foo" and "bar". The response would look similar to: 

{
    "foo": {
        "FileId": "50bc0ea3bf90af377fe01109"
    },
    "bar": {
        "FileId": "50afc470bf90af19c3ef3739"
    }
}

Note that these values are suitable for use as values for file attributes in POST/PUT requests to create/edit Audrey Objects.
